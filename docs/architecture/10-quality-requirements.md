# Quality Requirements

## Maintainability

- Backend, frontend, and build behavior should stay understandable for a small team.
- Technical decisions should be discoverable in versioned documentation.

## Developer experience

- Local setup should remain fast and explicit.
- The repository should support common workflows with a small number of commands.

## Deployability

- The application should remain packageable as a single unit.
- Infrastructure changes should remain reproducible from both a workstation and GitLab CI.

## Consistency

- Project version information should remain aligned across backend and frontend.
